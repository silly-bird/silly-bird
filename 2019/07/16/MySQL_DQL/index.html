聚合函数

聚合函数是用来做纵向运算的函数：

l COUNT()：统计指定列不为NULL的记录行数；

l MAX()：计算指定列的最大值，如果指定列是字符串类型，那么使用字符串排序运算；

l MIN()：计算指定列的最小值，如果指定列是字符串类型，那么使用字符串排序运算；

l SUM()：计算指定列的数值和，如果指定列类型不是数值类型，那么计算结果为0；

l AVG()：计算指定列的平均值，如果指定列类型不是数值类型，那么计算结果为0；

(2)　SUM和AVG

  • 当需要纵向求和时使用sum()函数。

    • 查询所有雇员月薪和：
      SELECT sum(sal) from emp

    • 查询所有雇员月薪和，以及所有雇员佣金和：

      SELECT sum(sal) 总工资,sum(comm) 总奖金 from emp

    • 查询所有雇员月薪+佣金和：

      SELECT empno, sum(sal+comm) from emp GROUP BY empno

    • 统计所有员工平均工资：

      SELECT empno, avg(sal) from emp GROUP BY empno

约束

作用：确保数据的完整性，保证用户添加到表中的数据是正确的

主键约束（PRIMARY KEY）

​ 保证每一行记录的是唯一的，并且是有效的

  • 特点：唯一性，非空.

  • 了解联合主键

  • 自增长（mysql独有，oracle使用序列的对象） auto_increment，注意其自动增长的值是从最后一次插入到值开始

  • uuid 用来作为主键来标识该行的唯一性

  • 第一种添加方式

第二种添加方式 方便添加联合主键

唯一约束(unique)

保证每一列记录的是唯一的，可以为null

思考：两个null和唯一有冲突吗？

​ 不冲突，语序有多个null

非空约束 （not null）

表明该字段不能为空

默认值约束（default）

当没有给该列赋值时，采用默认值

表与表之间的关系

一对一 ：一个人有一个身份证

  • 第一种在user表中引入对应身份证表的ID
  • 直接设为一张表（开发常用）

一对多：一个老师对应多个学生

  • 在多的那个表中引入对一这个表中id

外键约束

​ 保证从表中的关联值必须要是主表中存在的值

多对多：两个一对多，一个老师对应多个学生，一个学生也有多个老师

如何解决：需要创建第三张表,中间表中至少两个字段，这两个字段分别作为外键指向各自一方的主键.
